# 数据库课程设计需求概要与实现建议

## 1. 论文投稿系统

### 详细需求
**用户角色**: 作者、专家、编辑

**作者功能**:
- 个人信息管理: 姓名、年龄、所在单位、邮箱、学位、职称、籍贯、研究方向、个人简介、手机
- 单位管理: 可挂职多个单位(单位名称、城市、邮编)
- 论文投稿: 题目(中英文)、摘要(中英文)、关键词(中英文、多个)、论文附件(PDF/DOC)
- 作者管理: 多位作者、通讯作者指定、资助基金信息(项目名称、编号)
- 审稿进度查看: 任务描述、进度描述、完成日期、评审意见
- 修改论文: 在截止时间内修改论文信息和重新提交附件
- 版面费支付: 录用通知后按要求支付版面费

**专家功能**:
- 个人信息: 姓名、职称、单位、邮箱、联系电话、研究方向
- 审稿管理: 查看待审论文、提交评审意见(接受/小修/大修/拒绝)
- 评审内容: 正面评价、负面评价、修改意见
- 审稿费提现: 银行卡号、银行、开户名

**编辑功能**:
- 初审: 论文形式检查，内容完整性确认
- 送审: 选择3位研究方向相近专家，发送评审任务和论文
- 通知发送: 录用通知(含收款信息)、修稿通知(含截止时间)、拒稿通知
- 论文排期: 确定期号、卷号、页码

**业务规则**:
- 作者约2000人，专家约200人
- 每月新增稿件约100篇(10%接受、40%小修、30%大修、20%拒绝)
- 论文作者3-5人，最多送审3次

### 实现建议
- **核心表**: Users(用户角色表)、Papers(论文表)、Authors(作者表)、Units(单位表)、Reviews(审稿表)、Experts(专家表)、Funds(基金表)、Transactions(交易表)
- **推荐实现功能**: 论文投稿管理系统、专家审稿系统
- **技术要点**: 工作流引擎处理审稿流程、文件上传下载、邮件通知系统、支付接口集成

## 总体实现建议

### 技术选型
- **数据库**: MySQL/PostgreSQL
- **后端**: Spring Boot + MyBatis
- **前端**: Vue.js + Element UI
- **文件存储**: 阿里云OSS/七牛云

### 开发重点
1. **选择1-2个相对简单的系统进行完整实现**
2. **推荐选择**: 演出门票销售系统 + 宿舍管理系统
3. **核心功能**: 重点实现用户管理、主要业务流程、基础CRUD操作

### 数据库设计要点
1. 合理的主外键约束
2. 适当的索引优化
3. 数据完整性保证
4. 考虑并发访问控制

### 项目实施步骤
1. 需求分析与数据库设计
2. 核心表结构创建
3. 基础CRUD功能实现
4. 主要业务逻辑开发
5. 用户界面设计
6. 系统测试与优化
